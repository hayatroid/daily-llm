# /summarize 📊

## 目的

特定日の全記事を俯瞰的に分析し、その日の知的探求プロセス全体を「思考の軌跡」として描く日次サマリーを作成する

## 使い方

`/summarize [日付]`

## なぜ従来のサマリーは価値を失うのか 🎯

多くの日次サマリーは「今日の記事一覧」程度の価値しか提供していない：

- 記事1では○○について議論した
- 記事2では△△を実装した
- 記事3では□□を学んだ

確かに「何をしたか」は分かるが、**なぜその順序で思考が発展したのか**、**それらの間にどんな知的な連続性があったのか**は見えない。

### 失われている本質的価値

真に価値のある日次サマリーとは、その日の**知的探求の軌跡**を描くものではないだろうか？

- 最初はどんな疑問から始まったのか？
- その疑問がどのように発展・深化したのか？
- 複数の記事を通じて、どんな統合的理解に到達したのか？
- その日の探求が示す、より普遍的な洞察は何か？

### 「思考軌跡の俯瞰」という解決策

**目標**: その日の知的探求全体を一つの成長物語として描く

これは記事の「要素抽出→構造化」ではなく、**思考プロセスの発見と再現**である：

**核心原理**：

- 複数記事間での**思考の連続性・発展**を発見
- 一日を通じた**洞察の深化プロセス**を明らかに
- 個別記事では見えない**統合的な理解**を抽出
- 読者がその日の**知的成長を追体験**できる構成

## 実行手順 📋

### Phase 1: 知的探求の軌跡分析 (Intellectual Journey Analysis)

1. **記事群の俯瞰的読解**:

   - その日の全記事を時系列で通読
   - 各記事の問題意識を特定：「何に疑問を持ったのか？」
   - 記事間での話題の移行を観察：「なぜ次のトピックに関心が向かったのか？」
   - 思考の発展を追跡：「前の記事の洞察が次の記事にどう影響したか？」

2. **思考の連続性の発見**:

   - **具体的な質問を自分に問いかける**：
     - 「記事1の結論が記事2の出発点になっているか？」
     - 「異なるトピックでも同じ原理が適用されているか？」
     - 「一日を通じて一貫して追求していた本質的な問いは何か？」
   - **パターンの特定**：
     - 技術→哲学、具体→抽象、問題→解決策など
     - 同じ思考法が異なる分野で繰り返し使われている箇所

3. **統合的理解の発見**:
   - **「そういえば今日は...」の発見**：
     - 「今日は一日中、本質と表面の違いについて考えていた」
     - 「今日のテーマは『システムの構造設計』だった」
     - 「今日は『効率性の罠』という共通原理が見えた」
   - **より大きな原理への到達**：
     - 個別の技術的解決策を超えた設計哲学
     - 複数分野に適用可能な思考法則
     - その日の探求が示す普遍的洞察

### Phase 2: 成長物語の構築 (Growth Narrative Construction)

1. **思考の軌跡の物語化**:

   - その日の知的探求を**一つの発見プロセス**として構成
   - 各記事の位置づけを**物語の章**として整理
   - 疑問→探求→洞察→統合の**自然な流れ**を作る

2. **深層テーマの特定**:

   - 表面的なトピックを超えた**根本的な探求テーマ**
   - その日の思考を貫く**本質的な問い**
   - 複数記事が示す**統一された世界観**

3. **読者体験の設計**:
   - その日の知的成長プロセスを**追体験できる構成**
   - 記事群の相互関係が**自然に理解**できる展開
   - **「なるほど、こういう流れだったのか」**と感じられる物語

### Phase 3: サマリー執筆・統合 (Summary Creation & Integration)

1. **タイトルの協働決定**:

   - その日の探求の**本質**を表現する印象的なフレーズ
   - 「今日は○○」形式で15-40文字
   - 表面的なトピックではなく**深層テーマ**を表現

2. **思考軌跡サマリーの執筆**:

   - **「今日のユーザは」で始まる成長物語**として執筆
   - 各記事を**探求プロセスの一部**として位置づけ
   - その日の**知的成長の本質**を描く
   - 読者が**同じ発見プロセスを追体験**できる構成

3. **品質確保と統合**:
   - ファイル作成: `src/content/daily/YYYY-MM-DD/index.md`
   - 品質チェック: `npm run format && npm run lint`, `npm run textlint`

## /newプロセスへの統合 🔄

### 自動実行による改善

**/new実行時の最終段階で自動的にサマリー更新**：

1. **完全新規生成**: 既存index.mdは参照せず、その日の全記事から独立して生成
2. **思考軌跡の最新化**: 新しい記事を含めた全体の知的探求プロセスを再分析
3. **メタ洞察の進化**: 記事が増えるごとに深まる統合的理解を反映

### ダミーサマリーの廃止

- ダミーサマリーテンプレートは不要
- 常に完成されたサマリーが存在
- 手動実行の必要性を排除

## 思考軌跡分析の具体的技法 🔍

### 1. 時系列思考パターンの読解

**実際の分析例**：

- 記事1「TypeScript の型安全性」→ 記事2「Rust の所有権」→ 記事3「設計時の制約」
- **発見**：「コンパイル時にエラーを防ぐ」という共通アプローチ
- **洞察の受け継ぎ**：型安全性の議論が所有権理解の土台になり、それが設計哲学の一般論へ発展

**分析の観点**：

- 前の記事の**どの結論**が次の記事の**出発点**になったか？
- 思考が**質的に変化した瞬間**はどこか？（具体→抽象、技術→哲学）

### 2. 共鳴するテーマの発見

**発見の方法**：

- **キーワードの再登場**：「構造」「設計」「本質」などが複数記事で言及
- **同じ思考法の適用**：「なぜそうなるのか？」を複数分野で追求
- **対比構造の反復**：「表面 vs 本質」「効率 vs 価値」などの軸が一貫

**具体例**：

- 一見無関係な「CSS設計」と「API設計」で「制約による自由」という共通原理を発見
- 技術記事と哲学記事で「システム思考」が一貫して適用されている

### 3. 統合的理解の抽出

**「メタ発見」のパターン**：

- **方法論の発見**：「今日は一日中『なぜ』を3回繰り返していた」
- **価値観の明確化**：「効率性より本質性を重視していた」
- **思考の進化**：「朝は技術的だったが、夕方には哲学的になっていた」

**普遍的洞察の例**：

- 「良いシステムは制約から生まれる」
- 「表面的な解決は長期的な問題を生む」
- 「深い理解は複数分野の統合から生まれる」

## サマリー構成例 📄

```markdown
---
title: '今日は[深層テーマを表現する印象的フレーズ]'
tags: ['theme-1', 'insight-2', 'principle-3']
description: 'その日の知的探求プロセスの本質を1文で表現'
---

今日のユーザは、[最初の問題意識]から始まり、[思考の発展プロセス]を経て、[統合的理解]に到達した。

[第一段落: その日の探求の出発点と初期の洞察]

[第二段落: 思考の深化と複数分野への展開]

[第三段落: 最終的な統合的理解とメタレベルの洞察]
```

## よくある失敗パターン ❌

### 記事内容の機械的羅列

- **「記事1では○○、記事2では△△」**の列挙
- 各記事の**独立した要約**の並置
- **思考の連続性**を無視した構成

### 表面的なトピック整理

- **技術名や手法名**の羅列だけ
- **深層テーマ**への到達なし
- **メタレベルの洞察**の欠如

### 既存サマリーへの依存

- 前回サマリーの**表現や構成を流用**
- **新鮮な視点**の喪失
- **思考の進歩**を反映しない停滞

## 成功の判断基準 🎯

### 思考軌跡の明確性

- ✅ その日の知的探求の**全体的な流れ**が明確
- ✅ 各記事の**相互関係と発展性**が理解できる
- ✅ **個別記事を超えた統合的理解**が示されている

### 読者体験の質

- ✅ その日の**知的成長プロセスを追体験**できる
- ✅ 記事群の関係性が**自然に理解**できる
- ✅ **「こういう流れだったのか」**という発見がある

### 深層洞察の価値

- ✅ 表面的なトピック整理を超えた**本質的理解**
- ✅ その日の探求が示す**より大きな原理や法則**
- ✅ **知的成長の本質**への到達

## 重要原則 ⚠️

1. **独立した新規分析**

   - 既存サマリーに依存しない完全新規生成
   - その日の記事群から直接的な分析
   - 常に新鮮な視点での俯瞰

2. **思考プロセスの実演**

   - 要素の羅列ではなく成長物語として構成
   - 読者が発見プロセスを追体験できる展開
   - メタレベルの洞察への自然な導線

3. **自動化による品質向上**
   - /new実行時の自動更新で常に最新化
   - 手動実行の依存関係を排除
   - 一貫した高品質サマリーの維持

この手法により、その日の知的探求全体を俯瞰し、個別記事では見えない成長の軌跡を明らかにできる。

## 深い理解のために 📚

サマリー作成で行き詰まった時や、より深い原理を理解したい場合は `/.claude/philosophy/thoughtful-article-creation.md` を参照してください。記事作成と同様に、サマリー作成も「思考プロセスの実演」という共通原理で貫かれています。

また、このサマリー作成は通常 `/new` 実行時に自動実行されるため、手動での実行は特別な場合（過去のサマリー見直し等）に限定されます。
